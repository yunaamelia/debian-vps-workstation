# Agent Rules

This directory contains strict, modular rules that define the architecture, style, and behavior of the **Debian VPS Configurator** project. These rules are derived from the permanent project instructions.

## Rule System Organization

- **[structure.md](./structure.md)**: File placement, directory organization, and project layout.
- **[naming.md](./naming.md)**: Naming conventions for files, classes, functions, and variables.
- **[technology.md](./technology.md)**: Usage guidelines for the technology stack (CLI, Config, Validation).
- **[architecture.md](./architecture.md)**: Core architectural principles (4-Layer model, DI, Fail-safe).
- **[code-style.md](./code-style.md)**: Python coding standards, typing, and documentation.
- **[integration.md](./integration.md)**: Component interaction, layer boundaries, and extension rules.

## Usage

These rules are intended to be read by AI agents and developers to ensure all modifications adhere to the established project standards.
